# OpenAPI Documentation

Complete API specification, generation process, and validation strategies for ProcureFlow.

## Overview

ProcureFlow provides a complete OpenAPI 3.0 specification documenting all 13 API endpoints across 5 domains:

- **Catalog** - Item search and management
- **Cart** - Shopping cart operations
- **Checkout** - Purchase processing
- **Agent** - AI chat interface
- **Auth** - Authentication endpoints

The specification is manually maintained and served at `/api/openapi`. Future phases include automated generation from Zod schemas, CI validation, and breaking change detection.
